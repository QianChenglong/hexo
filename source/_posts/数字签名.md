# 名词

| 名词           | 解释             |
|----------------+------------------|
| 受信任的发送方 | 拥有密钥的发送方 |

# 过程

-   发送方

原文 => HASH => 加密(RSA) => 签名

-   接收方

除sign外的数据 => HASH ==? 发送过来的sign => 解密(RSA)

# 目的

-   保证数据是从指定源发送的

    因为受信任的发送方才有密钥，只有该密钥加密的数据才能正确地被接收方解密

-   防止数据在途中被第三方修改，保证了数据的完整性

    因为签名是基于全文HASH后得到的，任何修改原文操作，在没有密钥的情况下，无法重新签名，
    也就意味着该数据无效！
